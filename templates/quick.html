Quick Sort

What is Quick Sort?

QuickSort is a widely used sorting algorithm that follows the divide-and-conquer paradigm. It was developed by Tony Hoare in 1960. The basic idea behind QuickSort is to select a "pivot" element from the array and partition the other elements into two sub-arrays according to whether they are less than or greater than the pivot. The sub-arrays are then recursively sorted.

How Quick Sort works?

  1. Choose a Pivot:
    • Select a pivot element from the array. The pivot can be chosen in various ways, such as selecting the first element, the last element, the middle element, or even a random element.
  2. Partitioning:
    • Rearrange the array such that all elements less than the pivot are on its left, and all elements greater than the pivot are on its right. The pivot itself is in its final sorted position.
  3. Recursion:
    • Recursively apply the above steps to the sub-arrays formed by the partitioning process.

Example:

  • Select a pivot element. For this example, we'll choose the last element, which is 4.
    • Array: [5, 3, 7, 1, 8, 2, 6, 4] (Pivot: 4)
  • Rearrange the array so that elements less than the pivot are on the left, and elements greater than the pivot are on the right.
    • Array after partitioning: [3, 1, 2, 4, 8, 7, 6, 5] The pivot (4) is now in its sorted position.
  • Now, apply the QuickSort algorithm recursively on the left and right sub-arrays.
  • The sorted left sub-array, the pivot, and the sorted right sub-array combine to form the final sorted array.
    • Final sorted array: [1, 2, 3, 4, 5, 6, 7, 8]

Complexity Analysis:

Time Complexity:

  • Best Case: O(NlogN), when the pivot chosen at the each step divides the array into roughly equal halves.
  • Average Case: O(NlogN)
  • Worst Case: O(N2), When the array is already sorted and the pivot is always chosen as the smallest or largest element.

Space Complexity:

  • O(1), if we don't consider the recursive stack space. If we consider the recursive stack space then, in the worst case quicksort could make O(N).

Advantages:

  • In-place sorting.
  • It is a divide-and-conquer algorithm that makes it easier to solve problems.
  • It is efficient on large data sets.

Disadvantages:

  • It has a worst-case time complexity of O(N2), which occurs when the pivot is chosen poorly.
  • It is not a good choice for small data sets.
  • It is not a stable sort, meaning that if two elements have the same key, their relative order will not be preserved in the sorted output in case of quick sort, because here we are swapping elements according to the pivotâ€™s position (without considering their original positions).
