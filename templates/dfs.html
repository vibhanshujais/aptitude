DFS

What is DFS?

Depth-First Search (DFS) is another graph traversal algorithm used to explore all the vertices of a graph. Unlike BFS, DFS explores as far as possible along one branch before backtracking. It can be implemented using either recursion or an explicit stack.

How DFS algorithm works?

  1. Initialization:
    • Create a stack (or use recursion).
    • Mark all vertices as not visited.
  2. Choose a starting vertex:
    • Choose a starting vertex (source) and mark it as visited.
    • Push the starting vertex onto the stack (or use recursion).
  3. Explore neighbours:
    • While the stack is not empty (or while recursion continues), do the following:
      • Pop a vertex from the stack (or get the current vertex in the recursive call).
      • Process the popped vertex.
      • Push all adjacent vertices that haven't been visited yet onto the stack (or make recursive calls for unvisited neighbors).
      • Mark each newly pushed vertex as visited.
  4. Termination:
    • Continue the process until the stack is empty (or until recursion completes).

Example:

Consider this graph:

             A
            / \
           B   C
          / \   \
         D   E   F
  1. Initialization:
    • Stack: empty
    • Visited: [false, false, false, false, false, false]
  2. Starting Vertex:
    • Choose A as the starting vertex.
    • Push A onto the stack, mark A as visited.
  3. Explore neighbors:
    • Pop A, process A.
    • Push B and C onto the stack.
    • Visited: [true, false, false, false, false, false]
    • Pop B, process B.
    • Push D and E onto the stack.
    • Visited: [true, true, false, false, false, false]
    • Pop D, process D.
    • No new vertices to push.
    • Visited: [true, true, false, true, false, false]
    • Pop E, process E.
    • No new vertices to push.
    • Visited: [true, true, false, true, true, false]
    • Pop C, process C.
    • Push F onto the stack.
    • Visited: [true, true, true, true, false, false]
    • Pop F, process F.
    • No new vertices to push.
    • Visited: [true, true, true, true, true, true]
  4. Termination:
    • Continue until the stack is empty.

Complexity Analysis:

Time Complexity:

  • The time complexity of DFS is also O(V + E), where V is the number of vertices, and E is the number of edges.

Space Complexity:

  • The space complexity is O(V) due to the stack.

Advantages:

  • DFS is often preferred in certain scenarios, such as topological sorting, where the order of exploration matters.
  • Memory requirements can be lower compared to BFS in some cases.

Disadvantages:

  • It may not find the shortest path in unweighted graphs.
  • It may get stuck in infinite loops if not properly implemented or if the graph has cycles.
