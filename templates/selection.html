Selection sort

What is Selection Sort?

Selection Sort is simple sorting algorithm that works by repeatedly selecting the minimum element from the unsorted portion of the array and placing it at the beginning. The algorithm divides the array into two parts: the sorted part, which initially is empty, and the unsorted part, which contains all the elements.

How Selection Sort works?

  1. Initial State:
    • The entire array is considered unsorted.
    • The sorted part is initially empty.
  2. Finding the Minimum:
    • Iterate through the unsorted part of the array to find the minimum element.
    • Swap the minimum element with the first element of the unsorted part.
  3. Expanding the sorted part:
    • Expand the sorted part by moving its boundary one position to the right.
    • The moved element is now considered part of the sorted portion.
  4. Repeat:
    • Repeat steps 2 and 3 until the entire array is sorted.

Example:

Consider an unsorted array: [64,25,12,22,11]

  • Pass 1: [11, 25, 12, 22, 64] (Swapped 11 and 64)
  • Pass 2: [11, 12, 25, 22, 64] (Swapped 12 and 25)
  • Pass 3: [11, 12, 22, 25, 64] (Swapped 22 and 25)
  • Pass 4: [11, 12, 22, 25, 64] (No swaps needed)

Complexity Analysis:

Time Complexity:

  • O(N2) in every case.

Space Complexity:

  • O(N2)

Advantages:

  • Simple and easy to understand.
  • Works well with small datasets.
  • In-place sorting(doesn't require additional memory to store temporary data structures)

Disadvantages:

  • Time complexity of O(N2) in the worst and average case.
  • Does not work well on large datasets.
  • Does not preserve the relative order of items with equal keys which means it is not stable.
