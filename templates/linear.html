Linear Search

What is Linear Search?

Linear search, also known as sequential search, is a simple method for finding a specific value within a list. Imagine you have a list of items, and you want to find the position of a particular item in that list. Linear search checks each element of the list one by one until it finds the target element or reaches the end of the list.

How does Linear Search works?

  1. Start from the beginning.
    • Imagine you have a list of items, like a shopping list.
    • To find a specific item, you start at the beginning of the list.
  2. Check each element.
    • You go through each item one by one, starting from the first.
    • Compare each item with the one you're looking for.
  3. Stop when you find it.
    • If the current item matches what you're looking for, you stop searching.
    • You note the position or index of the item in the list.
  4. End of list
    • If you go through the entire list and don't find the item, you know it's not there.

Example:

Consider the following list: [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5], and you want to find the position of the number 2.

  1. Start from the beginning of the list.
  2. Check the first element: 3 - not a match.
  3. Move to the next element: 1 - not a match.
  4. Next: 4 - still no match.
  5. Continue this process until you find 2.

Complexity Analysis:

Time Complexity:

  • Best Case:In the best case, the key might be present at the first index. So the best case complexity is O(1)
  • Worst Case:In the worst case, the key might be present at the last index i.e., opposite to the end from which the search has started in the list. So the worst-case complexity is O(N) where N is the size of the list.
  • Average Case:O(N)

Space Complexity:

  • O(1) as except for the variable to iterate through the list, no other variable is used.

Advantages:

  • Linear search can be used irrespective of whether the array is sorted or not. It can be used on arrays of any data type.
  • Does not require any additional memory.
  • It is a well-suited algorithm for small datasets.

Disadvantages:

  • has a time complexity of O(N), which in turn makes it slow for large datasets.
