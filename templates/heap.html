Heap Sort

What is Heap Sort?

Heap Sort is a comparison-based sorting algorithm that uses a binary heap data structure to organize elements for efficient sorting. It was first proposed by J.W.J. Williams in 1964. The algorithm has a time complexity of O(n log n) for the worst, average, and best cases, making it relatively efficient.

How Heap sort works?

  1. Heapify the array:
    • The first step is to build a max-heap from the given array. A max-heap is a binary tree where the value of each node is greater than or equal to the values of its children. The idea is to start from the last non-leaf node and move upwards to the root, ensuring that the heap property is maintained at each step.
  2. Build Max-Heap:
    • Starting from the last non-leaf node (which is at index (n/2 - 1) in a zero-based index system, where n is the number of elements in the array), perform a heapify operation to maintain the max-heap property. Continue this process until you reach the root.
  3. Extraxt Max and Heapify:
    • Now, the max-heap is built, and the largest element (at the root) can be extracted. Swap this element with the last element in the array and reduce the heap size by one. After the swap, heapify the root to maintain the max-heap property.
  4. Repeat:
    • Repeat the extraction and heapify steps until the heap size becomes 1. After each extraction, the largest remaining element is placed at the end of the array.
  5. Sorted array:
    • Once all elements are extracted, the array is sorted in ascending order.

Complexity Analysis:

Time Complexity:

  • O(nlogn) in every case.

Space Complexity:

  • O(1)

Advantages:

  • Heap Sort is an in-place sorting algorithm, which means it uses only a constant amount of extra memory for swapping elements during the sorting process.
  • Heap Sort is a stable sorting algorithm.
  • Heap Sort has good cache performance, making it suitable for external sorting scenarios where data is too large to fit into the computer's main memory.

Disadvantages:

  • Heap Sort does not adapt to the existing order of elements in the input array.
  • Slower on Small Datasets.
