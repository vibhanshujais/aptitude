Binary Search

What is Binary Search?

Binary search is a widely used algorithm for finding a specific target value within a sorted array or list. It follows the "divide and conquer" strategy, where the search space is repeatedly divided into smaller parts until the target is found or the search space is empty.

How does Binary Search works?

  1. Precondition:
    • Binary search requires the input array or list to be sorted in ascending or descending order. This is crucial for the algorithm to work correctly.
  2. Initialization:
    • Set two pointers, low and high, initially pointing to the first and last elements of the array, respectively.
    • Calculate the middle index as mid = (low + high) / 2.
  3. Comparison:
    • Compare the middle element (arr[mid]) with the target value.
    • If the middle element is equal to the target, the search is successful, and the index of the target is found.
    • If the target is less than arr[mid], adjust the high pointer to mid - 1, and repeat the process in the lower half of the array.
    • If the target is greater than arr[mid], adjust the low pointer to mid + 1, and repeat the process in the upper half of the array.
  4. Update Midpoint:
    • Recalculate the middle index as mid = (low + high) / 2 after adjusting the pointers.
  5. Repeat:
    • Continue the process until the target is found or the low pointer surpasses the high pointer. This means the target is not present in the array.

Complexity Analysis:

Time Complexity:

  • Best Case: O(1)
  • Average Case: O(log N)
  • Worst Case:O(log N)

Space Complexity:

  • O(1), If the recursive call stack is considered then the auxiliary space will be O(logN).

Advantages:

  • Binary search is faster than linear search, especially for large arrays.
  • More efficient than other searching algorithms with a similar time complexity, such as interpolation search or exponential search.
  • well-suited for searching large datasets that are stored in external memory, such as on a hard drive or in the cloud.

Disadvantages:

  • The array should be sorted.
  • equires that the data structure being searched be stored in contiguous memory locations.
