Counting Sort

What is Counting Sort?

Counting Sort is a non-comparative sorting algorithm that works by counting the number of occurrences of each element in the input. It assumes that the input consists of integers within a specific range. Counting Sort has a linear time complexity, making it very efficient for certain types of data.

How Counting Sort works?

  1. Determine the range:
    • Find the range of input elements. This is necessary because Counting Sort works well when the range of input elements is known in advance.
  2. Initialize Coutn array:
    • Create an auxiliary array called the Count array to store the count of each unique element. The size of this array is determined by the range of input elements.
  3. Count Occurrences:
    • Traverse the input array and increment the corresponding count in the Count array for each element encountered.
  4. Calculate Cumulative Counta:
    • Modify the Count array to store cumulative counts. This step helps determine the final position of each element in the sorted output.
  5. Build the Output Array:
    • Create an output array of the same size as the input array. Traverse the input array again, and for each element, place it in its correct sorted position in the output array based on the cumulative count.
  6. Decrement Counts:
    • Decrement the count in the Count array for each element placed in the output array. This step is essential for handling duplicate elements.

Example:

Consider the input array: [4, 2, 1, 4, 3, 1]

  • The range is determined to be 4 (max element is 4).
  • Create an array of size 5 (range + 1) initialized with zeros: [0, 0, 0, 0, 0].
  • Traverse the input array and increment the count in the Count array for each element encountered.
    • Count array after counting occurrences: [0, 2, 1, 1, 2]
  • Modify the Count array to store cumulative counts.
    • Count array after calculating cumulative counts: [0, 2, 3, 4, 6]
  • Create an output array of the same size as the input array.
  • Traverse the input array again and place each element in its correct sorted position in the output array based on the cumulative count.
    • Output array: [1, 1, 2, 3, 4, 4]

Complexity Analysis:

Time Complexity:

  • O(N+M), in every case.

Space Complexity:

  • O(N+M), N and M are the space taken by count array and output array.

Advantages:

  • Linear time complexity
  • Effective when sorting integers within a specific range.
  • Stable sorting

Disadvantages:

  • Note in place sorting means it requires additional memory to store the count and sorted output
