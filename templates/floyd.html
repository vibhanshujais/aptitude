Floyd-Warshall Algorithm

What is Floyd-Warshall Algorithm?

The Floyd-Warshall algorithm is an algorithm for finding the shortest paths between all pairs of vertices in a weighted graph, including graphs with negative weight edges. It does so by considering all possible intermediate vertices along the paths. The algorithm was proposed by Robert Floyd in 1962 and independently by Stephen Warshall in 1962.

How does Floyd-Warshall Algorithm works?

  1. Initialization:
    • For each pair of vertices i and j, initialize the distance matrix D such that D[i][j] is the weight of the edge from vertex i to vertex j if there is an edge, and âˆž otherwise.
    • Set D[i][i]=0 for all vertices i.
  2. Main Loop:
    • For each intermediate vertex k, iterate through all pairs of vertices i and j and update D[i][j] if the path from i to j through k is shorter than the current value of D[i][j].
    • The update rule is: D[i][j]=min(D[i][j],D[i][k]+D[k][j]).
  3. Result:
    • After the algorithm completes, the matrix D will contain the shortest distances between all pairs of vertices.

Complexity Analysis:

Time Complexity:

  • The time complexity of the Floyd-Warshall algorithm is O(V3 ) where V is the number of vertices in the graph.

Space Complexity:

  • The space complexity is O(V2 ) for storing the distance matrix.

Advantages:

  • Handles both positive and negative weight edges.
  • Can handle graphs with negative cycles.
  • Suitable for dense graphs where the number of edges is close to V 2.

Disadvantages:

  • Inefficient for large graphs due to its cubic time complexity.
  • Typically not as fast as specialized algorithms (e.g., Dijkstra's or Bellman-Ford) for specific cases.
