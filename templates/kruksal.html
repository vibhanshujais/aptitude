Kruksal's Algorithm

What is Kruksal's Algorithm?

Kruskal's algorithm is another greedy algorithm used for finding the minimum spanning tree (MST) of a connected, undirected graph. The key idea is to sort the edges in ascending order of their weights and then add edges to the MST one by one, ensuring that adding an edge does not create a cycle. This algorithm uses a disjoint-set data structure to keep track of the connected components of the graph.

How Kruksal's Algorithm works?

  1. Sort edges:
    • Sort all the edges of the graph in non-decreasing order of their weights.
  2. Initialize Disjoint sets:
    • Create a disjoint-set data structure to keep track of connected components.
  3. Iterate over edges:
    • Iterate through the sorted edges and add each edge to the MST if it does not create a cycle.
    • The disjoint-set data structure helps in checking whether adding an edge creates a cycle.
  4. Termination:
    • Continue this process until the MST has n-1 edges, where n is the number of vertices in the graph.

Complexity Analysis:

Time Complexity:

  • The overall time complexity is dominated by the sorting step, resulting in O(ElogE) time complexity.

Space Complexity:

  • The space complexity is O(V+E), where V is the number of vertices and E is the number of edges.

Advantages:

  • Kruskal's algorithm produces the optimal MST for a connected, undirected graph with weighted edges.
  • It is efficient, especially when dealing with sparse graphs where E is much less than V 2 .

Disadvantages:

  • Kruskal's algorithm may not be as efficient for dense graphs, where the number of edges is close to V 2
  • The disjoint-set data structure requires additional space, making the space complexity relatively higher.
