BFS

What is BFS?

BFS stands for Breadth-First Search, which is a graph traversal algorithm used to explore all the vertices of a graph in breadthward motion. It starts at the source vertex and explores its neighbors before moving on to the next level of neighbors. BFS is often used to find the shortest path in an unweighted graph.

How BFS algorithm works?

  1. Initialization:
    • Create a queue to keep track of the vertices to be visited.
    • Mark all vertices as not visited.
  2. Choose a starting vertex:
    • Choose a starting vertex (source) and mark it as visited.
    • Enqueue the starting vertex into the queue.
  3. Explore neighbours:
    • While the queue is not empty, do the following:
      • Dequeue a vertex from the queue.
      • Process the dequeued vertex.
      • Enqueue all adjacent vertices that haven't been visited yet.
      • Mark each newly enqueued vertex as visited.
  4. Termination:
    • Continue the process until the queue is empty.

Example:

Consider this graph:

             A
            / \
           B   C
          / \   \
         D   E   F
  1. Initialization:
    • Queue: empty
    • Visited: [false, false, false, false, false, false]
  2. Starting Vertex:
    • Choose A as the starting vertex.
    • Enqueue A, mark A as visited.
  3. Explore neighbors:
    • Dequeue A, process A.
    • Enqueue B and C.
    • Visited: [true, false, false, false, false, false]
    • Dequeue B, process B.
    • Enqueue D and E.
    • Visited: [true, true, false, false, false, false]
    • Dequeue C, process C.
    • Enqueue F.
    • Visited: [true, true, true, false, false, false]
    • Dequeue D, process D.
    • No new vertices to enqueue.
    • Visited: [true, true, true, true, false, false]
    • Dequeue E, process E.
    • No new vertices to enqueue.
    • Visited: [true, true, true, true, true, false]
    • Dequeue F, process F.
    • No new vertices to enqueue.
    • Visited: [true, true, true, true, true, true]
  4. Termination:
    • Continue until the queue is empty.

Complexity Analysis:

Time Complexity:

  • The time complexity of BFS is typically expressed as O(V + E), where V is the number of vertices (nodes) and E is the number of edges in the graph.

Space Complexity:

  • The space complexity of BFS is O(V) for the worst-case scenario, where V is the number of vertices.

Advantages:

  • BFS is well-suited for finding the shortest path in an unweighted graph
  • BFS is complete, meaning it will find a solution (if one exists) in a finite amount of time, given that the graph is finite.
  • BFS typically requires less memory because it only needs to store information

Disadvantages:

  • BFS does not consider edge weights. It is not suitable for finding the shortest path in graphs with weighted edges
  • In sparse graphs, where the number of edges is significantly less than the number of vertices, BFS may explore many vertices that are not on the shortest path.
